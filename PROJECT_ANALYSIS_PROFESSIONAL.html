Purpose: Complete product catalog stored in client-side memory

Contains: 100+ product objects with properties: id, category, name, price, oldPrice, image, description, brand, color, inStock

Categories: laptops, phones, headphones, watches, accessories

category.js (100 lines)

Renders category-specific product listings by:

  • Determining current category from page title
  • Initializing filter and sort modules
  • Rendering product grid dynamically
  • Handling responsive filter panel toggle

Advanced filtering system with:

  • Price Range: Dual inputs + range sliders
  • Brand: Checkboxes for each brand
  • Color: Visual color swatches with hex codes
  • Availability: "In Stock Only" checkbox
sort.js (120 lines)

Sorting functionality with 6 options:

  1. Default (original order)
  2. Price Low to High
  3. Price High to Low
  4. Name A to Z
  5. Name Z to A
  6. In Stock First

Data Storage Locations

  • MongoDB Atlas Cloud: User accounts, authentication, profile data
  • Browser LocalStorage: Shopping cart items (persists across sessions)
  • JavaScript Array: Product catalog (100+ items, loaded on page load)

User Schema Structure

  _id: ObjectId,                    // MongoDB auto-generated
  first: String (2+ chars),         // First name, required
  last: String (2+ chars),          // Last name, required
  email: String (unique, regex),    // Validated email format
  password: String (6+ chars),      // Hashed with bcryptjs
  nick: String,                     // Optional nickname
  gender: String,                   // Optional
  language: String,                 // Optional
  country: String,                  // Optional
  timezone: String,                 // Optional
  address: String,                  // Optional delivery address
  created: Date,                    // Auto timestamp
  createdAt: Date,                  // Mongoose timestamp
  updatedAt: Date                   // Mongoose timestamp
}

API Endpoints

  • GET / - Serve home page
  • POST /api/register - User registration (201 on success, 400/409/503 errors)
  • POST /api/login - User login (200 on success, 401/503 errors)
  • PUT /api/profile - Update user profile (200 on success, 400/404/503 errors)
  • GET /api/users - Get all users (200 on success, 503 error)
